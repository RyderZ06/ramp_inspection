Чтобы привязать локальную папку проекта к Git-репозиторию и создать удалённый репозиторий на GitHub для отслеживания изменений, выполните следующий подробный алгоритм:

1. Инициализация локального репозитория Git
В терминале (в вашем виртуальном окружении или обычном cmd) перейдите в папку проекта:

text
cd путь\к\проекту
Инициализируйте Git-репозиторий:

text
git init
Это создаст скрытую папку .git и сделает текущую папку репозиторием.

2. Добавление файлов и первый коммит
Добавьте все файлы проекта в индекс Git:

text
git add -A
Сделайте первый коммит с сообщением:

text
git commit -m "Initial commit"
3. Создание удалённого репозитория на GitHub
Перейдите на GitHub и войдите в свой аккаунт.

Нажмите кнопку New repository (Новый репозиторий).

Введите имя репозитория (обычно совпадает с именем проекта).

Можно добавить описание, выбрать публичный или приватный репозиторий.

Нажмите Create repository.

4. Привязка локального репозитория к удалённому
В терминале в папке проекта выполните команду, подставив URL вашего репозитория с GitHub (он будет вида https://github.com/username/repo.git):

text
git remote add origin https://github.com/username/repo.git
Проверьте, что удалённый репозиторий добавлен:

text
git remote -v
Вы увидите что-то вроде:

text
origin  https://github.com/username/repo.git (fetch)
origin  https://github.com/username/repo.git (push)
Если возникнет ошибка, что origin уже существует, можно удалить старый remote:

text
git remote remove origin
и повторить добавление.

5. Отправка изменений на GitHub
Отправьте локальные коммиты в удалённый репозиторий:

text
git push -u origin master
(или, если у вас основная ветка называется main, замените master на main)

При первом пуше вас попросят ввести логин и токен доступа GitHub (пароли через HTTPS больше не используются, вместо них - personal access token).

Итог
Теперь ваш локальный проект связан с удалённым репозиторием на GitHub. Все изменения можно отслеживать, коммитить и отправлять на сервер. В дальнейшем для пуша достаточно будет просто:

text
git push
Если хотите, могу помочь с созданием personal access token, настройкой веток или автоматизацией через GitHub CLI. Хотите?
